# /review

変更コードをレビューする。

## 手順

1. 差分を確認する
   ```
   git diff
   ```

2. 以下の観点でレビューし、問題点を指摘する

   ### コード品質
   - CLAUDE.md のコーディング規約に沿っているか
   - `any` 型を乱用していないか
   - 関数・クラスにコメントが書かれているか
   - コードが機能単位のブロックに分割されているか

   ### 設計
   - REST の原則に沿った API 設計になっているか
   - routers / services / repositories の責務分離が守られているか
   - Pydantic による入力バリデーションが行われているか

   ### セキュリティ
   - SQLインジェクション等の脆弱性がないか
   - `.env` の機密情報がコードにハードコードされていないか

   ### テスト
   - `python -m pytest tests/` を実行してテストがすべて通るか確認する

3. 問題がなければ承認、問題があれば修正箇所を具体的に示す
